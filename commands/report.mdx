---
title: /report Command
description: Report bugs or issues with ScryxCLI
---

The `/report` command allows you to report bugs, issues, or problems you encounter while using ScryxCLI.

## Usage

```bash
/report
```

## How It Works

<Steps>
  <Step title="Enter the command">
    Type `/report` in the chat and press `Enter`.
  </Step>
  
  <Step title="Read the information">
    You'll see a message explaining the bug report process:
    
    ```
    🐞 Uh-oh! Found a bug in SCRYCLI?
    We'd really appreciate your help in squashing it
    Your feedback helps make SCRYCLI better for everyone 💜❤️
    Click [Enter] to report the issue on our website. 🛠️
    ```
  </Step>
  
  <Step title="Confirm">
    Press `Enter` to open the bug report form in your default web browser.
  </Step>
  
  <Step title="Submit your report">
    The ScryxCLI bug report page opens at:
    
    ```
    https://scrycli.tech/report
    ```
    
    After the browser opens, you'll see:
    
    ```
    👋 Thanks for reporting! We'll get back to you soon.
    ```
  </Step>
</Steps>

## What to Include in Your Report

When reporting a bug, include:

- **Description**: What happened and what you expected
- **Steps to reproduce**: How to trigger the bug
- **Environment**: Your OS, ScryxCLI version, etc.
- **Error messages**: Any error text or screenshots
- **Context**: What you were trying to do when the bug occurred

## Navigation

- **Enter**: Open the bug report form
- **Esc**: Cancel and return to chat

## Example Flow

```bash
> /report
[Esc] to go back

🐞 Uh-oh! Found a bug in SCRYCLI?
We'd really appreciate your help in squashing it
Your feedback helps make SCRYCLI better for everyone 💜❤️
Click [Enter] to report the issue on our website. 🛠️

> [Enter]

# Browser opens to https://scrycli.tech/report

👋 Thanks for reporting! We'll get back to you soon.
```

<Note>
  The report form opens in your default web browser. Make sure you have a browser configured and can access the internet.
</Note>

## Common Issues to Report

- Application crashes or freezes
- Unexpected error messages
- Commands not working as documented
- Performance problems
- Incorrect AI responses or behavior
- Installation or setup issues
- Documentation errors or unclear instructions

## Privacy

When you use `/report`, only your web browser is opened. No automatic data collection or telemetry is sent. You control what information to share in the report form.

## Response Time

The ScryxCLI team reviews bug reports regularly. While response times vary, you can expect:

- Acknowledgment of critical bugs: Within 24-48 hours
- Updates on bug fixes: As patches are developed
- General reports: Response within a few days

## Alternative Reporting Methods

Besides `/report`, you can also:

- Check the GitHub repository for known issues
- Contact support through official channels
- Join the community discussion forums
