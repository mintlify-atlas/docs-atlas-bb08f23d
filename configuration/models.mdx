## Overview

ScryxCLI supports multiple AI providers and models for code analysis. You can select your preferred model through an interactive CLI interface or by manually editing the configuration file.

## Available Providers

ScryxCLI supports the following AI providers through OpenRouter:

| Provider | Models Available | Recommended Model |
|----------|------------------|-------------------|
| OpenAI | 2 models | gpt-oss-120b |
| ArceeAI | 2 models | Trinity Large |
| Qwen | 2 models | Qwen3 Next 80B A3B Instruct |
| Mistral | 1 model | Mistral Small 3.1 24B |
| StepFun | 1 model | Step 3.5 Flash |
| Zai | 1 model | GLM 4.5 Air |

<Note>
  All models listed are free-tier models accessible through OpenRouter with rate limits.
</Note>

## Selecting a Model

### Using the CLI Command

<Steps>
  <Step title="Open model selection">
    While ScryxCLI is running, type:
    
    ```bash
    /model
    ```
  </Step>
  
  <Step title="Choose your provider">
    Use arrow keys to navigate and Enter to select:
    
    ```
    > OpenAI
      StepFun
      Qwen
      ArceeAI
      Mistral
      Zai
    ```
  </Step>
  
  <Step title="Select a model">
    After choosing a provider, select from available models:
    
    ```
    > gpt-oss-120b (Recommended)
      gpt-oss-20b
    ```
  </Step>
  
  <Step title="Configuration saved">
    Your selection is automatically saved to `~/.scrycli/config.json`
  </Step>
</Steps>

## Configuration Structure

When you select a model, ScryxCLI updates your configuration:

```json
{
  "model": {
    "modelProvider": "openai",
    "modelName": "openai/gpt-oss-120b:free",
    "modelKey": "your-api-key"
  }
}
```

The configuration is set via:

```typescript
// From src/ui/SelectModel.tsx:68
setConfig('model', {
  modelProvider: selectedProvider,
  modelName: item.value
});
```

### Configuration Fields

| Field | Description | Example |
|-------|-------------|----------|
| `modelProvider` | Provider identifier | `"openai"`, `"qwen"`, `"arceeai"` |
| `modelName` | Full model path | `"openai/gpt-oss-120b:free"` |
| `modelKey` | Your API key | `"sk-or-v1-..."` |

<Warning>
  You must set `modelKey` separately. The model selection command only sets `modelProvider` and `modelName`.
</Warning>

## Model Selection Process

The model selection UI follows a two-step process:

```typescript
// From src/ui/SelectModel.tsx:44-75
const SelectModel = (props: selectModelType) => {
    const [selectedProvider, setSelectedProvider] = useState<string | null>(null);
    const [selectedModel, setSelectedModel] = useState<string | null>(null);

    // Step 1: Select provider
    if (!selectedProvider) {
        return <SelectInput items={providers} onSelect={setSelectedProvider} />;
    }

    // Step 2: Select model from provider's available models
    if (selectedProvider && !selectedModel) {
        return <SelectInput 
            items={modelsMap[selectedProvider]} 
            onSelect={(item) => {
                setSelectedModel(item.value);
                setConfig('model', { 
                    modelProvider: selectedProvider, 
                    modelName: item.value 
                });
            }} 
        />;
    }
};
```

The selected model is used when making AI calls:

```typescript

const result = openRouterClient.callModel({

```

## Validation Requirements

Before ScryxCLI can make model calls, it validates:

```typescript
// From src/lib/isModelSelected.ts:6-15
const isModelSelected = (): boolean => {
    const config = getConfig();
    if (!config?.model?.modelName) return false;
    if (!config?.model?.modelKey) return false;
    return config.model.modelName !== '' && config.model.modelKey !== '';
};
```

Both `modelName` and `modelKey` must be:
- Present in the configuration
- Non-empty strings

## Troubleshooting

### Model Not Selected Error

If you see "Model: Not Selected":

<Steps>
  <Step title="Run model selection">
    ```bash
    /model
    ```
  </Step>
  
  <Step title="Verify configuration">
    ```bash
    cat ~/.scrycli/config.json
    ```
    
    Ensure `model.modelName` exists
  </Step>
  
  <Step title="Add API key">
    Make sure `model.modelKey` is also set
  </Step>
</Steps>

### Model Calls Failing

If model calls fail:

1. **Check model name format**
   - Must be in format: `provider/model:tier`
   - Example: `openai/gpt-oss-120b:free`

2. **Verify API key**
   - See [API Keys Configuration](/configuration/api-keys)

3. **Check OpenRouter status**
   - Some models may be temporarily unavailable
